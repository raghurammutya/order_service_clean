# Legacy Code Files

This directory contains legacy code files that are no longer used in the current implementation but are preserved for historical reference.

## ⚠️ WARNING

**DO NOT USE THESE FILES IN PRODUCTION**

These files contain outdated patterns, deprecated configurations, or superseded implementations. They are kept solely for:
- Historical reference
- Audit trail purposes  
- Understanding migration decisions

## Directory Structure

### `/config/`
Legacy configuration files:
- Pre-compliance configuration patterns
- Backup configurations from migrations
- Deprecated settings structures

## Current Implementation

For current code implementations, refer to:
- `app/config/settings.py` - Current configuration
- `app/` - Active application code
- `docs/ARCHITECTURE_COMPLIANCE.md` - Current architecture patterns

## Usage Guidelines

### ✅ Appropriate Uses
- Understanding historical implementation decisions
- Audit and compliance reference
- Learning about migration patterns

### ❌ Inappropriate Uses  
- Copy-pasting into current code
- Reference for new implementations
- Production deployment
- Integration with current systems

## Maintenance

- These files should NOT be modified
- New legacy files should include clear deprecation warnings
- Consider periodic review for relevance (every 12 months)
